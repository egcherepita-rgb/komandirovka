# Календарь командировок — FastAPI

Рабочая версия сайта, где **единственным источником данных является сам сайт и база SQLite**.
Excel больше не нужен для ежедневной работы.

## Что умеет
- Показывает матрицу бронирования по неделям и товарным направлениям
- Позволяет создавать новые слоты прямо на сайте
- Позволяет менять статус слота
- Позволяет указывать филиал, контакт, цель визита, приоритет и комментарий
- Даёт список всех слотов с фильтрами
- Позволяет удалить ненужный слот
- Сохраняет данные в SQLite

## Запуск
```bash
cd travel_calendar_fastapi
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Открыть в браузере:
- http://127.0.0.1:8000

## Где лежат данные
- База: `data/calendar.db`

## Логика работы
- После запуска сайт работает автономно
- Все изменения вносятся только через веб-интерфейс
- Excel не участвует в обычной работе
- При пустой базе сайт покажет стартовый экран и предложит создать первый слот

## Разовый перенос данных из старого Excel
Если нужно один раз перенести старые данные из XLSX, можно выполнить импорт вручную:

```bash
python -m app.import_xlsx "data/Календарь_командировок_2026 (1).xlsx"
```

Это именно разовый перенос. После него дальнейшая работа ведётся только на сайте.
